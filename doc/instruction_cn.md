# 自动化库存与智能零售 Agent

---

## 1. 项目目标

该项目的目标是构建一个自主运行的零售 Agent，利用计算机视觉技术实时审计物理货架。通过检测商品、空缺和错放，该 Agent 能够消除数字数据库记录与门店实际物理现状之间的信息差。

### “幽灵库存”

如依靠传统的数据库仅依靠销售数据进行记录，引起三个盲区：

* **商品错放：** 考虑顾客把商品随手丢在其他货架上，该商品对真正想买它的顾客而言不可见。数据库中，它依然在库，但实际上销量已归零。Agent 能识别出商品位置错误，并通知员工将其归位。
* **盗窃与商品损耗：** 被盗或损坏的商品不会经过结账扫描。这些商品在数据库中依旧在库，因此不会触发补货预警。Agent 则能发现物理货架已经为空，从而标记异常并通知重新订货。
* **补货延迟：** 数据库可能显示某种商品还有大量库存，但实际上这些商品堆积在仓库。该项目前端可展示货架已空，Agent 发现货架变空后，会核对库房数据，并提醒员工将库存搬到前台货架。

---

## 2. AI Agent 工作流程

系统通过以下连续的三步循环运行：

1. **感知 (Perception)：** 目标检测模型处理货架的视频流或照片，定位商品和空缺空间。
2. **推理 (Reasoning)：** AI Agent 将视觉检测结果与实时数据库或数字化门店布局进行比对。
3. **行动 (Action)：** Agent 自动执行相应操作，如自动生成供应商采购订单、给卖场员工发送通知或根据供需动态调整价格。

---

## 3. 实现

由于商品数目种类繁多，本项目将存在极端的“多类别分类”问题。如果想让模型直接去分辨成千上万种具体的品牌，完成难度极大。相反，该项目采用以下高效的、基于坐标的“空缺检测 (Gap Detection)”方案。

### I.数据集：RP2K

该项目采用高质量、干净的开源零售数据集 RP2K，通过迁移学习来避免从零开始训练模型。

### II.训练双类别“空缺检测”模型

该项目并非让模型去区分数百种不同的品牌（例如可乐 vs 百事），而是训练一个像 **YOLOv8** 这样的轻量化目标检测架构，让它只检测以下两种状态：

1. `Product`（商品）
2. `Empty Shelf Space (Gap)`（货架空缺）

### III.建立数字化货架位图 (Planogram)

创建门店布局的数字化地图（即陈列图/货架位图）。该数据库将具体的物理货架坐标直接与应当放置的商品绑定。

* *示例：* `坐标 (X: 12, Y: 45) = Y 品牌苏打水`

### IV.基于坐标进行逻辑推理

编写 AI Agent 代码，将视觉模型的输出结果与你的货架位图坐标结合起来，从而推断出缺失的库存：

1. 计算机视觉模型检测到某个特定坐标处存在 `Empty Shelf Space (Gap)`（货架空缺）。
2. AI Agent 在数字化货架位图中查询该精确坐标。
3. Agent 做出推理：*“坐标 (X:12, Y:45) 处出现空缺。根据位图，这里应该摆放 Y 品牌苏打水。因此，Y 品牌苏打水目前处于缺货状态。”*
